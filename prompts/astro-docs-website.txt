You are an Expert Frontend Engineer and Elite UI/UX Designer.

Build a premium, blazing-fast, and highly responsive landing page and documentation website for "Git Wrangler".

Goal:
Create an Astro-based landing page and documentation hub for an open-source, bash-based CLI tool designed for orchestrating complex Git operations across dozens of repositories simultaneously.

Rules:
- strictly adhere to all design decisions, aesthetics, component styles, color tokens, spacing rules, typography, and interaction patterns described in `DESIGN.md`
- do NOT use Tailwind CSS; use Vanilla CSS exclusively
- build a robust design system using CSS Custom Properties (variables) aligned with token definitions in `DESIGN.md`
- use Astro (latest version) and initialize the project in a new `website/` directory
- implement Astro's native `<ClientRouter />` API for SPA-like transitions
- use component-driven architecture with reusable `.astro` components
- err on the side of more components rather than fewer
- ensure responsive and logical navigation (sticky top navbar, collapsible mobile-friendly sidebar)
- integrate `logo.svg` from the project root into Navbar and Footer/Hero
- use Astro Content Collections (`src/content/docs/`) for documentation
- dynamically generate the left-hand navigation sidebar from Content Collection frontmatter
- group documentation pages into specific categories: `Utility`, `History Rewriting`, `Local Operations`, `Remote Operations`, and `General`

Process:
1. Scaffold the Astro project inside a new `website/` subdirectory.
2. Update `website/package.json` to include the required `@fontsource` dependencies.
3. Establish base layouts (`BaseLayout.astro`, `DocsLayout.astro`) with `<ClientRouter />` and global CSS variables.
4. Build UI components (Navbar, Footer, Sidebar, CodeBlock).
5. Assemble the Landing Page (`/`) with a Hero Section and Features Section.
6. Parse metadata headers from `libexec/wrangler-*` scripts to create `.md` files for Command Reference documentation.
7. Create initial Markdown pages (Introduction, Installation, Architecture) extracting copy from `README.md`.
8. Assemble Docs pages using the dynamically generated sidebar based on frontmatter categories.

Output:
Create a fully functional Astro website codebase strictly inside the `website/` directory.

Do not deviate from or extend the design system defined in `DESIGN.md`.
Do not pollute the CLI project root with website code.
Do not require the user to touch an `.astro` or configuration file to add a new command to the documentation.
