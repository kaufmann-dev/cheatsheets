Create a production-ready install.sh script for a GitHub project.

The script must work across macOS, Linux, and Windows (via Git Bash or WSL).

REQUIREMENTS:

1. Installation method
- Must support:
  curl -fsSL https://raw.githubusercontent.com/<owner>/<repo>/main/install.sh | bash
- Must be safe to run in a pipe (no interactive prompts unless absolutely necessary)

2. OS + architecture support
- Detect OS: macOS, Linux, Windows (Git Bash / WSL)
- Detect architecture: amd64, arm64 (and fallback handling)
- Download/install correct binary or dependency based on system

3. Reliability + safety
- Must be idempotent (safe to run multiple times)
- Must not corrupt existing installs
- Include basic error handling (missing curl, permission issues, failed download)
- Fail fast with clear error messages

4. Curl install best practices
- Analyze and implement the most robust curl-based installation pattern
- Prefer secure download methods (e.g., version pinning, optional checksum verification if applicable)
- Avoid unnecessary external dependencies
- Optimize for speed and reliability

5. Output behavior
- Script should print clear status messages (installing, detected OS, success, etc.)
- Keep output minimal but informative

6. README update requirement
After generating the install logic, update README.md to include:
- Installation section
- The curl install command:
  curl -fsSL https://raw.githubusercontent.com/<owner>/<repo>/main/install.sh | bash
- Short explanation of what the installer does
