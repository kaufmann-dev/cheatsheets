You are an expert Bash developer and Git internal specialist.

Create a Bash script that rewrites commit timestamps across all managed Git repositories.

Goal:
Redistribute commit timestamps across a date range so the history looks like natural human activity rather than a bulk rewrite.

Rules:
- use `git-filter-repo` to rewrite commit dates
- do NOT use `git filter-branch` under any circumstances
- verify `git-filter-repo` is available on the system at the start
- place the script in the `libexec/` directory without a `.sh` extension
- include a standardized header block (`Usage`, `Description`, `Category`, `Options`)
- use a `while IFS= read -r` loop fed by a here-string of collected paths to iterate through repositories
- wrap each iteration in an isolated subshell
- accept `--start-date` and `--end-date` flags in `YYYY-MM-DD` format
- format output with project standard ANSI color codes (`\e[31m`, `\e[32m`, `\e[33m`, `\e[0m`)
- always pass variables as arguments to `printf` using `%s`, never pipe directly into format strings

Process:
1. Discover all target `.git` directories using `find`.
2. Implement stratified jitter: divide the date range into N slots and place each commit with up to 40% jitter.
3. Implement bimodal time-of-day weighting: sample hours with peaks at 10am and 3pm, clamped between 7am and 10pm.
4. Infer timezone offset from the most common offset in the commit log, formatting timestamps accordingly for `GIT_AUTHOR_DATE` and `GIT_COMMITTER_DATE`.
5. Implement weekend dampening: nudge 65% of weekend commits to Friday evening or Monday morning.
6. Implement micro-clustering: space same-day commits 25 to 90 minutes apart.
7. Sort all assigned timestamps in ascending order and apply them to commits in original topological order.
8. Print a summary showing old and new dates for each commit and require a `--confirm` flag or interactive prompt before rewriting.

Output:
Provide the Bash script implementation.

Do not pipe variables directly into printf format strings.
Do not proceed without user confirmation or the `--confirm` flag.
