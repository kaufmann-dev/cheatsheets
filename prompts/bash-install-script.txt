You are an expert Bash developer and DevOps engineer.

Create a production-ready install.sh script for a GitHub project.

Goal:
Build a robust, multi-platform installation script that safely installs the project via a `curl | bash` pipe.

Rules:
- the script must work across macOS, Linux, and Windows (via Git Bash or WSL)
- must be safe to run in a pipe (no interactive prompts unless absolutely necessary)
- detect OS and architecture (amd64, arm64) and handle fallbacks
- must be idempotent (safe to run multiple times without corrupting existing installs)
- include basic error handling (missing curl, permission issues, failed downloads)
- fail fast with clear error messages
- use secure download methods (e.g., version pinning, optional checksum verification)
- avoid unnecessary external dependencies
- keep output minimal but informative (e.g., installing, detected OS, success)

Process:
1. Implement OS and architecture detection logic.
2. Verify required dependencies (like `curl`) are available.
3. Set up safe error handling (`set -e`, fail fast).
4. Download and install the correct binary or dependency based on the system.
5. Print clear, formatted status messages during the installation steps.
6. Update the `README.md` to include an Installation section with the `curl` command.

Output:
Provide the `install.sh` script and the `README.md` addition.

Do not prompt for interactive input.
Do not assume `root` privileges unless strictly necessary.
